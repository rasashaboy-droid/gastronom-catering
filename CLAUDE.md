# Gastronom catering — кейтеринг в Казани. Инструкции для продолжения работы

Этот документ — передача проекта.

## Что это

Лендинг для кейтеринг-сервиса «Gastronom Catering» (Казань). Один HTML-файл с внешними JSX-скриптами, без сборки — всё компилируется в браузере через `@babel/standalone`.

**Стек:** React 18 UMD + Babel standalone. Никаких npm, vite, tsconfig. Файлы открываются напрямую в браузере.

## Структура

```
Landing.html         ← корневой файл: <head> со всеми стилями, <body> с #root, подключает jsx
app.jsx              ← <Nav>, <App>, root.render
hero.jsx             ← <Hero>, <HeroSplit> (ТОЛЬКО split-вариант, bento/stack удалены)
components.jsx       ← <Placeholder>, <Chip>, <SecLabel>, <Blob>, Icon.{Arrow, Spark, Phone, ...}
sections.jsx         ← все остальные секции: StatsBand, Formats, Advantages, QuizCalc,
                       MenuHighlights, Process, FAQ, FinalCTA, Footer
                       (компонент Social определён, но НЕ рендерится — удалён из app.jsx)
uploads/             ← исходники пользователя + референсные скрины (pasted-*.png)
```

Порядок подключения скриптов в `Landing.html` (НЕ менять):
```
components.jsx → hero.jsx → sections.jsx → app.jsx
```
Каждый следующий файл зависит от предыдущих. Все компоненты — в глобальной области (Babel в браузере не даёт скопов между `<script type="text/babel">`), поэтому НЕ добавляй `export` / `import`.

## Цветовая палитра (CSS переменные в Landing.html)

```

--yellow:          #FFEB69   редкий яркий акцент (логотип, плитка "Красиво", stat-карточка)
--yellow-soft:    #FFF4A8

--cream-50:  #FFF6EC   фон сайта
--cream-100: #FFEADA
--cream-200: #FFD9C2
--cream-300: #FFEB69   (совпадает с --yellow; исторический алиас)

--peach:  #FFD9C2
--coral:  #F4557A
--tomato: #E91E4C    (алиас --raspberry)
--brick:  #B8143A

--ink: #1A0A0F       основной текст
--ink-60/40/15/08    прозрачности ink
```

**Правила использования цвета:**
- Основной фон — `--cream-50`. Секции НЕ получают свой фон без необходимости.
- Малиновый (`--raspberry`) — главный акцент: кнопки primary, italic-вставки (`em.accent-italic`), ключевые числа, квиз-карточка.
- Жёлтый — акцент-редкость: логотип-иконка, плитка «Красиво» в Advantages, радиал-декор в квизе. Не заливай им большие плоскости.
- Кремовые оттенки — для карточек-плиток, пилюль в Formats, тегов, placeholder'ов.
- Никогда не добавляй новые HEX-ы вне этой палитры. Если нужен новый оттенок — используй `color-mix()` или oklch, основанный на существующих переменных.

## Типографика

- **Display** (заголовки): `Unbounded`, weights 600/700/800. CSS-класс `.display` + часто `fontFamily: 'Unbounded, sans-serif'` инлайном (чтобы гарантированно перебить body).
- **Body**: `Inter`, 400/500/600/700.
- **Mono** (мелкие подписи, tech): `JetBrains Mono`.

Все три — Google Fonts, подключены в `<head>` Landing.html одним `<link>`.

**italic-акценты:** внутри h1/h2 используется `<em className="accent-italic">...</em>` — малиновый, italic, weight 600. Это ключевой визуальный приём: цитата в референсе «Кейтеринг в Казани *от 600 ₽*/чел.».

**Размеры:** везде через `clamp(min, fluid, max)`. Пример: `fontSize: 'clamp(34px, 4.2vw, 60px)'`. НЕ используй фиксированные px для display-текста.

## Дизайн-язык

- **Bento / асимметричные сетки.** В Advantages — CSS Grid 12 колонок, карточки занимают 5/4/3 сверху и 6/6 снизу. В Social, MenuHighlights — тоже grid-based.
- **Стеклянные поверхности.** Классы `.glass` и `.glass-dark` — backdrop-filter + полупрозрачный фон. Используй для nav, поверх цветных подложек.
- **Пилюли.** Кнопки, чипы, nav-ссылки — `border-radius: 999px`. Углы квадратных карточек — 20–36px (через clamp).
- **Placeholder'ы.** Для всех фото — компонент `<Placeholder label="..." variant="peach|coral|cream"/>`. Никогда не пиши реальные `<img>` — бренд пока без ассетов, всё через placeholder с диагональной штриховкой.
- **Иконки.** Только из `Icon.*` в components.jsx (stroke 1.6, единый стиль). Новые иконки добавляй туда же, такой же толщиной.
- **Кнопки.** Три класса: `.btn.btn-primary` (малиновый заполненный), `.btn.btn-ghost` (прозрачный с бордером), `.btn.btn-light` (кремовый). Все с hover: `translateY(-2px)`.

## Адаптив

В Landing.html есть большой блок с `@media` правилами — breakpoints:
- `980px` — hero split → 1 колонка
- `900px` — Formats список → 1 колонка, фото идёт сверху
- `1040px` / `640px` — Advantages bento reflow
- `860px` — Quiz calc (отдельный мобильный лейаут, см. ниже)
- `720px` — Nav links скрываются
- `560px` — Nav burger, hero photo → квадрат
- `480px` — калькулятор опции

**Мобильный квиз-калькулятор (особый случай).**
На `<860px` розовая карточка и бежевая становятся ДВУМЯ отдельными карточками одинаковой ширины, с перекрытием:
- `.calc-shell` теряет свой розовый фон (становится прозрачной обёрткой)
- Левая колонка (`.calc-split > div:first-child`) получает розовый фон + border-radius 28px
- `.glass` — отдельная бежевая карточка ниже
- Розовая перекрывает бежевую: `margin-bottom: -20px`, `z-index: 2`
- Бежевая компенсирует через `padding-top: 44px`
- Декоративные gloss/radial скрыты (`.calc-shell > div:nth-child(1|2) { display:none }`)

Если трогаешь квиз — сохрани эту логику, она выверена с пользователем.

**Mobile Advantages reflow (важно):** DOM-порядок детей в `.adv-bento` = [01 Вкусно, 02 Красиво, 04 Надёжно, 03 Индивидуально, 05 Безопасно] (items[0], items[1], items[3], items[2], items[4]). Это НЕ ошибка — так нужно, потому что на мобилке `:nth-child(2)` и `:nth-child(3)` делаются квадратами 2×1 (Красиво + Надёжно). Не меняй порядок рендера без обновления media-правил.

## Технические запреты

- **НЕ добавляй `scrollIntoView`** — есть прецедент поломки scroll в превью-хосте. Используй `window.scrollTo({top: el.offsetTop - offset, behavior: 'smooth'})`.
- **НЕ добавляй `export`/`import`/`type="module"`.** JSX компилируется браузером, модули не поддерживаются.
- **НЕ создавай файлы `const styles = {...}`** в глобале JSX — коллизии имён. Если нужен объект стилей, называй его `heroStyles`, `calcStyles` и т.д.
- **НЕ используй новые HEX-цвета** — только переменные из палитры выше.
- **НЕ трогай `Landing.html` как bundle** — пользователь один раз сделал standalone-версию, она поломала базовый файл (React+ReactDOM загружались после скомпилированного кода). Если нужен автономный HTML — сохраняй его отдельным именем (`Landing-standalone.html`), не перезаписывай `Landing.html`.
- **НЕ перекомпилируй JSX в inline script** в Landing.html. Сохраняй ссылки на внешние `.jsx`.
- **НЕ добавляй спикер-ноты, декрипта, meta-теги для экспорта** — это лендинг, не слайды.

## Логика работы, которой я следовал

1. **Референсы → палитра → шрифты.** Пользователь даёт скрины (в `uploads/`), я извлекаю цвета и шрифты, прописываю в переменные в `<head>` Landing.html. Только потом трогаю компоненты.
2. **Инлайн-стили для точечных правок, CSS для структурных.** Когда нужно поменять цвет одного элемента — инлайн. Когда адаптив или поведение на hover — класс + CSS в Landing.html.
3. **Класс как якорь для media query.** Для каждой сетки, которая должна ломаться на мобилке, добавляю уникальный класс (`.hero-split`, `.formats-split`, `.adv-bento`, `.calc-shell`, `.calc-split`, `.calc-opts`, `.social-stats`, `.social-reviews`, `.process-grid`, `.faq-split`, `.cta-split`). Правила media только в Landing.html, НЕ в JSX.
4. **Один источник правды для палитры.** Все цвета из `:root` в Landing.html. В JSX — только через `var(--...)`. Если видишь захардкоженный HEX — это баг.
5. **Placeholder-первый подход.** Пока нет фотосессии, всё через `<Placeholder>`. Когда клиент предоставит ассеты — заменить точечно, не трогая layout.
6. **Итерации через короткие правки.** Пользователь говорит конкретное — я делаю минимальное изменение. Не рефактори «заодно», не добавляй секции без запроса.

## Как проверять

1. Открывай `Landing.html` в превью.
2. Ресайзь окно — смотри все breakpoints (980, 900, 860, 720, 560, 480).
3. Проверяй консоль — ошибок быть не должно. Часто ломается порядок загрузки скриптов или опечатки в JSX.
4. Квиз, калькулятор гостей (slider), FAQ (accordion), mobile-menu — интерактив, проверяй клики.
5. Если нужен скриншот в узкой ширине — делай iframe в тестовом HTML с `width="390"` (обычный CSS resize не триггерит media queries в превью).
