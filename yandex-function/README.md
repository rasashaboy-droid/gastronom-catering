# Yandex Cloud Function: заявки → Telegram

Функция принимает POST-запросы с форм сайта и шлёт сообщение в Telegram.
Токен бота и chat_id хранятся только в переменных окружения — в коде секретов нет.

## Деплой — пошагово

### 1. Зайди в Yandex Cloud Console

<https://console.cloud.yandex.ru/>

Создай каталог (папку), например `gastronom-leads`, если ещё не создал.

### 2. Создай Cloud Function

1. Слева в меню каталога → **Cloud Functions**
2. Нажми **Создать функцию**
3. Имя: `gastronom-leads` (или любое). Нажми **Создать**

### 3. Создай версию функции (код + env vars)

На странице функции нажми **Создать в редакторе**.

Заполни:

- **Среда выполнения:** `nodejs18` (или `nodejs20`, если доступна)
- **Способ:** `ZIP-архив` или **Редактор кода**
- **Точка входа:** `index.handler`
- **Таймаут:** `5` секунд
- **Память:** `128 МБ` (хватит)

**Код:**
В редакторе создай файл `index.js` и вставь содержимое из `yandex-function/index.js` этого репозитория.

**Переменные окружения** (внизу страницы, раздел «Переменные окружения»):

| Имя              | Значение |
|------------------|----------|
| `BOT_TOKEN`      | токен от BotFather (полный, например `8670779369:AAH-...`) |
| `CHAT_ID`        | твой chat_id (например `7827918649`) |
| `ALLOWED_ORIGINS`| `https://rasashaboy-droid.github.io` (через запятую можно добавить свой домен позже) |

**Важно:** вкладка переменных помечена как «секретные» — галочка рядом с `BOT_TOKEN`, чтобы значение не отображалось в интерфейсе. Это сохранит токен в зашифрованном виде.

Нажми **Сохранить изменения**.

### 4. Сделай функцию публичной (HTTPS-вызов)

1. На странице функции вкладка **Обзор**
2. Найди раздел **Публичная функция** → включи тумблер
3. Скопируй появившийся **ID функции** — URL будет такой:
   ```
   https://functions.yandexcloud.net/<ID_ФУНКЦИИ>
   ```

Этот URL я пропишу в код фронтенда.

### 5. Проверь, что работает

В терминале (подставь свой URL):

```bash
curl -X POST https://functions.yandexcloud.net/<ID_ФУНКЦИИ> \
  -H "Content-Type: application/json" \
  -H "Origin: https://rasashaboy-droid.github.io" \
  -d '{"source":"quote-modal-price","name":"Тест","phone":"+79990001122","page":"/"}'
```

Ожидаемый ответ: `{"ok":true}`.
В Telegram должно прилететь сообщение.

Если вернулась ошибка — проверь в Yandex Cloud → Функция → вкладка **Логи**.

## Что уже настроено в коде функции

- CORS — разрешены только origin'ы из `ALLOWED_ORIGINS`
- Rate-limit — 5 заявок с одного IP за минуту
- Honeypot-поле `hp_field` — если заполнено, запрос молча игнорируется (защита от ботов)
- Валидация: телефон минимум 10 цифр
- Поля обрезаются до разумной длины (против DoS)
- Время в МСК

## Если хочется большей защиты позже

- Yandex SmartCaptcha (бесплатно, нативно интегрируется)
- Persistent rate-limit через Yandex Database (YDB)
- API Gateway вместо прямого вызова функции (гранулярный rate-limit, кастомный домен)
